# 通知警告演示

## 描述

当通知功能不可用时，脚本显示的警告界面和处理逻辑。

## 场景

用户配置的通知方式不可用时，显示警告并询问是否继续。

## 警告界面

```
========================================
⚠️  通知功能不可用
========================================

当前配置的通知方式: url
问题:
  - 未配置 REMOTE_NOTIFY_URL
  - 或缺少 HTTP 客户端 (curl/wget)

⚠️  注意事项:

  如果您是通过 远程 SSH 连接到此主机:
    - 修改 MAC 后 IP 地址可能会改变
    - 您需要使用其他方式找到主机的 IP 地址
    - 可用方法:
      • 路由器 DHCP 客户端列表
      • 扫描局域网: sudo ./linux-mac-changer.sh scan 192.168.70.0/24
      • 直接连显示器查看 IP

  如果您是在 本地控制 此主机:
    - 无需担心，修改后可直接在终端看到新 IP
    - 本地文件通知将保存到: /tmp/new_ip.txt

💡 建议: 可以使用 'localfile' 模式作为保底方案

========================================

是否继续？(y/N):
```

## 用户选择

### 取消操作

显示配置指南:

```
已取消操作
如需继续，请配置通知功能后重试:

方法1: 使用本地文件通知
  编辑脚本，设置: NOTIFY_METHOD="localfile"

方法2: 配置 URL 通知
  1. 启动通知服务器: python3 notification-server.py
  2. 编辑脚本，设置: REMOTE_NOTIFY_URL="http://YOUR_IP:8089"
```

### 继续执行

```
✓ 继续执行（通知功能已禁用）
  MAC 修改功能将正常工作
  已自动切换到 localfile 通知模式
```

## 检测逻辑

### NOTIFY_METHOD="url"

**可用条件**:
- REMOTE_NOTIFY_URL已配置
- curl或wget可用

**不可用条件**:
- URL未配置
- 缺少HTTP客户端

### NOTIFY_METHOD="telegram"

**可用条件**:
- TELEGRAM_BOT_TOKEN已配置
- TELEGRAM_CHAT_ID已配置
- curl可用

**不可用条件**:
- Token或ChatID未配置
- 缺少curl

### NOTIFY_METHOD="all"

**可用条件**:
- 以上任一方式可用

**不可用条件**:
- 所有方式都不可用

## 自动降级

如果用户选择继续:
1. NOTIFY_METHOD自动设置为"localfile"
2. 保证MAC修改功能正常工作
3. 本地文件通知始终可用

## 设计理念

1. **用户友好**: 清晰说明问题和风险
2. **场景区分**: 区分远程SSH和本地控制
3. **提供方案**: 给出具体解决方案
4. **尊重选择**: 让用户决定是否继续
5. **自动降级**: 继续时自动切换到安全模式
