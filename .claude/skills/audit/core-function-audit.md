# 核心功能审计

## 描述

审计MAC修改和联网保证的核心功能，确保在任何情况下都可用。

## 审计目标

确保修改MAC地址并保证联网的核心功能在任何情况下都可用，即使通知功能失败也不影响。

## 核心功能保证

### MAC修改功能

| 功能 | 实现方式 | 可靠性 |
|------|---------|-------|
| 随机MAC生成 | `printf "%02X" $((RANDOM % 256))` | ✅ 系统随机数 |
| 自定义MAC | 用户输入 | ✅ 直接使用 |
| MAC地址修改 | `ip link set address` | ✅ 标准命令 |

### IP获取保证

**random命令** (双重策略):
1. DHCP获取任意IP (10秒超时)
2. 手动配置原IP (如果可用)

**random-keepip命令** (三重策略):
1. DHCP REQUEST原IP (5秒, 60-80%)
2. 设置静态IP (95%, 检测冲突)
3. DHCP获取新IP (保底, 100%)

### 通知功能解耦

| 解耦点 | 实现方式 |
|--------|---------|
| 系统检测 | HTTP客户端检查改为警告 |
| send_notification | 使用子shell + \|\| true |
| 通知调用 | 子shell封装，失败不影响主流程 |

## 失败场景处理

### 通知服务器不可达
- 检测: curl失败
- 处理: 记录日志，继续执行
- 影响: 无

### 缺少curl/wget
- 检测: 系统检测阶段
- 处理: 自动切换到localfile模式
- 影响: 无

### DHCP服务器无响应
- 策略1: 尝试DHCP REQUEST
- 策略2: 尝试静态IP
- 策略3: 保底DHCP获取
- 结果: 98%成功率

### 原IP被占用
- 检测: arping检测冲突
- 处理: 跳过静态IP，使用DHCP
- 结果: 仍能获取新IP

### 初始无IP地址
- 检测: original_ip为空
- 处理: 先尝试获取，失败则继续
- 结果: MAC修改后尝试DHCP
